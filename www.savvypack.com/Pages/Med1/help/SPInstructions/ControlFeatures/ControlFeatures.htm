Section III:

Control Features

A.  Global Manager

The Global Manager is the first screen you see after selecting and starting a module on the Universal Manager screen.  It contains the highest level control features, and you should leave it open during your entire session.  If you close the Global Manager, you will no longer have the ability to change and to manage your cases.

There are two primary courses of action available to you on the Global Manager screen.  You can start a case or start the Toolbox.  These and other actions are described in more detail in the following paragraphs.

1.  Starting Cases

One of the most important features provided by the Global Manager is the ability to start a case for review and analysis.  As described previously, you can start two types of cases – base cases and proprietary cases.  

Starting a base case

Begin by selecting a case from the drop down menu located in the gray box labeled “Select from Available Base Cases.”  Once you have selected the base case you’d like to start, submit your choice by clicking the “Start a Base Case” button just below the dropdown menu.  When you start a case in this manner, the Case Manager appears in a new browser window with all the links necessary to examine your selected case.

Starting a proprietary case

Begin by selecting a case from the drop down menu located in the second gray box from the top labeled “Select from Available Proprietary Client Cases.”  Once you have selected the proprietary case you’d like to start, submit your choice by clicking the “Start a Proprietary Case Analysis” button just below the dropdown menu. When you start a case in this manner, the Case Manager appears in a new browser window with all the links necessary to examine your selected case.

2.  Starting the Toolbox

The Global Manager also provides access to the Toolbox. The Toolbox contains tools for high level case control including creating new cases, copying existing cases, deleting cases, sharing cases, synchronizing cases, and others.

To start the Toolbox, scroll to the bottom of the Global Manager screen and find the button labeled “Go to Toolbox”.  Click this button to open the Toolbox.  The Toolbox will open in a new browser window.

3.  Starting the Case Viewer

The Case Viewer is provided to assist with case selection. The Case Viewer provides a detailed description of each case in a convenient-to-browse format.

Starting Base Case Viewer

Begin by finding the link called “Case Viewer” in the gray base case box.  Clink this link to open the Case Viewer, which provides a detailed description of each base case.   The Case Viewer opens in a new browser window.

 

Starting Proprietary Case Viewer

Begin by finding the link called “Proprietary Case Viewer” in the grey proprietary case box.  Clink this link to open the Case Viewer, which provides a detailed description of your proprietary cases.   The Case Viewer opens in a new browser window.

 

 

B.  Toolbox

The Toolbox provides tools for key case management functions.  These include creating cases, copying cases, and other functions that are described below.  In addition, step by step procedures are provided for each Toolbox function in the next section of the manual.

1.  Copy function

The copy function creates a new case (the target case) in your proprietary space then copies all of the data from another case (the source case) into the new target case.  You select the source case, which can be a base case or one of your proprietary cases.  The system automatically creates and numbers the new target case.

Once the copy function has completed, a new window will open with a message verifying that the copy completed successfully and providing the case ID number of the new proprietary case.

2.  Transfer function

The transfer function transfers all of the variables from a source case to an existing target case.  The transfer function is similar to the copy function in that the copy and transfer functions move the same set of variables from one case to another.  You select both the source and the target case for the transfer function.

Once the transfer function has completed, a new window will open with a message verifying that the transfer completed successfully and identifying the source and target case numbers.

You can use the transfer function to transfer data to a target case that is blank (a brand new case) or one that is completely populated with data.  In either situation, the data from the source case will be transferred to the target case.  As a result, you MUST double check the target case you have selected to ensure you do not copy over an existing case that you wish to keep.

3.  Create function

The create function creates and automatically numbers a new proprietary case.  All of the data, including the case name and description, will be blank or zero for this new case.

Once the create function has completed, a new message window will open verifying that a new proprietary case was created successfully and providing you with the number of the new case.

4.  Delete function

The delete function deletes the proprietary case that you have selected.  As a result, you MUST double check the target case you have selected to ensure you do not delete an existing case that you wish to keep.

Once the case deletion function is completed, a new message window will open verifying that the proprietary case was deleted successfully.

5.  Rename

The rename function provides the capability to change the name and description of proprietary cases.  You select the target case.

Once the rename function is finished updating, a new message window will open verifying that the case descriptions were successfully updated and identifying the case effected.

6.  Share function

The share function provides the capability to share access to a proprietary case or to share a copy of a proprietary case with a colleague.  You can share your proprietary cases with other licensed users at your company.

 

Share access

The share access function allows you to share access to your proprietary cases.  Shared access means all users are accessing the same case.  Changes made by one user to a case shared by access are changed for all users.

Once the share access function has completed, a new message window will open verifying that the case has been shared successfully and identifying the user that received access.

Share copy

The share copy function allows you to share a copy of your proprietary cases with other licensed users at your company.  Shared copy means all users receive their own copy of the source case.  Changes made by one user to a case shared by copy are changed in their copy only.

Once the share copy function has completed, a new message window will open verifying that the case has been copied successfully and identifying the user that received the copy.

7.  Synchronize function

The synchronize function transfers variables from one module to another.  However, the synchronize function is carefully and automatically controlled to limit synchronization to transfers that make sense.  It is possible to synchronize between Econ1 and Sustain1 modules.  It is also possible to synchronize cases between Econ2 and Sustain2 modules.  However, it is not possible to synchronize between Econ1 and Econ2 modules or Sustain1 and Sustain2 modules.

  The synchronize function is also limited to transferring data between cases having the same case ID number.  Whenever a new case is created in one module, a corresponding case having the same case ID number is automatically created in the module that can be synchronized.  For example, when a new proprietary case is created in Econ1, a corresponding proprietary case having the same case ID number is created in Sustain1.

Once the synchronization function has completed, a new window will open verifying that you have successfully synchronized and identifying the case number and the source and target modules.

 

 

C.  Case Manager

The Case Manager screen is the key screen for reviewing and examining individual cases.  The Case Manager screen is the first screen that comes up after starting a case from the Global Manager screen. The Case Manager screen’s features are organized into four categories:

•  Assumptions screens

•  Intermediate results screens

•  Final results screens

•  Case Preferences screen

1.  Assumptions screens

Depending on which module you are working in, there are fourteen or fifteen assumption screens that are accessed from the Case Manager.  The links to the Assumptions screens are found on the left hand column of the Case Manager screen under the “Specify Assumptions” heading. To access an assumption screen, click on the assumption page link you are interested in and that assumption page will open in a new window.  When you are done working in the assumption page, close the window to return to the Case Manager.  Note that each assumption page must be accessed and filled out correctly in order for the system to calculate results. The instructions for filling out each assumption page are provided in later sections of this manual.

 It is generally not going to be necessary for you to fill out all of the assumptions on each assumption page.  SavvyPack Corporation provides pre-researched Base Cases that you can copy into your proprietary space using the copy tools in the Toolbox.  Using this strategy, you then have a case in your proprietary space with the assumptions pages filled out.  At this point, you only need to modify the assumptions necessary to create a case that models the specific package or process you are interested in.

 

2.  Intermediate results screens

From the Case Manager, you can also access intermediate results pages.  The intermediate results pages serve two primary purposes.  First, they create transparency within the SavvyPack@ System by showing intermediate results, which make it easy to understand the underlying calculations.  Second, they allow you to gauge whether the assumptions you have entered are reasonable based on the intermediate results.  Specific instructions on how to use the intermediate results pages are found in later sections of this manual.

The intermediate results pages are found in the middle column of the Case Manager screen under the heading “Review Intermediate Results”.  To access an intermediate results page, click the link and the page will open in a new window.  When you are done with intermediate results page, close the window to return to the Case Manager screen.

3.  Final results screens

From the Case Manager screen, you can also access the final results pages.  The final results pages are the primary outputs of the SavvyPack system.  For the Econ1 and Econ2 modules, the final results pages consist of a pro-forma profit and loss sheet with or without depreciation, a cost sheet with or without depreciation, and a case summary page that summarizes key assumptions and results for printing.  For Sustain1 and Sustain2, the final results pages consist of energy consumption, greenhouse gas releases, end-of-life, scorecard results, and a case summary page.  The final results pages also include a link to a compare feature that makes it possible to compare cases, both in tables and with graphs.

To access the Final Results pages, click the link to the results page you are interested in located beneath the “Review Final Results” heading.  The final results page you select will open in a new window.  When you are done with the page, close the window to return to the Case Manager screen.   Specific instructions on how to use each Final Results page are provided in later sections of this manual.

4.  Case Preferences screen

From the Case Manager screen, you can also access the Case Preferences page.  The Case Preferences page is important because you make selections on the Case Preferences page that impact the entire case.  On the Case Preferences page you select the effective date for material prices, energy rates, labor rates, and currency conversion rates.  You also select the units (Metric or English) and the currency you wish to work in.  You also select the country of manufacture.

To access the Case Preferences page, click the “Preferences” link above the “Review Intermediate Results” heading.  The Case Preferences page opens in a new window.  Once you are done with the Case Preferences page, close the window to return to the Case Manager screen.  Specific instructions for the Case Preferences page are provided in later sections of this manual.
