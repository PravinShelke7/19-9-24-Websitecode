Section II: 

User Guide Introduction

The Introduction to the User Guide explains the organization of the User Guide and the organization of the SavvyPack® System itself.

A.  User Guide organization

The User Guide is organized in several sections:

•  Introduction

The introduction begins with the organization of the SavvyPack® System.

 

B.  SavvyPack® System organization

The SavvyPack® System is organized into four modules.  The user interfaces with each module consists through a series of web pages.  The modules and the hierarchy of web pages are examined in this sub-section.

1.  Modules

The SavvyPack® System is divided into modules.  There are four modules that form the SavvyPack® Engine, the heart of the SavvyPack® System.  

The four modules of the SavvyPack® Engine are distinguished by the type of analysis they perform (economic or environmental) and by their position in the supply chain (package production or product packaging).  The SavvyPack® Engine modules are defined as the following:

•  Econ1 module – economic analysis of package production

•  Econ2 module – economic analysis of product packaging

•  Sustain1 module – environmental impact analysis of package production

•  Sustain2 module– environmental impact analysis of product packaging

The four modules  of the SavvyPack® Engine form a completely integrated system with communication links and data transfer between modules.  

2.  Hierarchy

Entry into the SavvyPack® System is obtained by logging into the member entry page.

The Universal Manager is the first page of the SavvyPack® System.  From the Universal Manager, the user selects the module to enter.  The next screen is the Global Manager for the module selected.  The organization of each module is examined in the following paragraphs.

 

C.  Module organization

Each module in the SavvyPack® Engine has two distinct spaces.  One space is for working on a specific case and the second is for high-level case management.

1.  Specific case organization

 A case is an individual analysis, and each analysis is based on a specific product and specific manufacturing process.

The cases within each module include two types of cases – base cases and proprietary cases.  Base cases are researched and entered into the SavvyPack® System by SavvyPack Corporation, and are then made available to all users.  The base cases cannot be modified, except by SavvyPack Corporation personnel.

Proprietary cases are developed by users.  Proprietary Cases are available only to the user that created them, unless that user takes the steps necessary to share them with a colleague.  

2.  Case management organization

Case management is conducted in the “Toolbox” of each module.  As just mentioned, it may be desirable to share a case with a colleague, and this is done in the Toolbox.  The Toolbox includes case management features such as creating a new case, importing data from one case to another, copying cases, deleting cases, sharing cases, and synchronizing cases.  All of these functions are explained in detail in later sections of this manual.

 

D.  Software and content updates

SavvyPack Corporation updates the software and content in the SavvyPack® system on a regular basis.  New software features and new content are added automatically, without requiring action from the user.

New software features and new content are announced to users on the Universal Manager page.  When announcements are made, a “SavvyPack® Announcements” link appears at the top of the Universal Manager page.  Users can click the link then read the announcements.  If all the announcements are acknowledged by the user, the SavvyPack® Announcements link will disappear.  If announcements are not acknowledged by the user, they remain available until they are automatically deleted, which is triggered six months from the date when the announcement was originally posted.
